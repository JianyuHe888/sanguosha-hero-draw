# 面杀助手：递进推荐将池与技能助手设计

## 背景与目标

现有“推荐将池”用一个布尔字段一次性纳入标准、神话再临、一将成名、全部界限突破和早期八神，共 219 名武将。这个规则有两个问题：

1. 移动版会调整武将分包和技能，同一个大类中的武将并不一定属于同一出版阶段或强度区间。
2. SP 老将被整体排除，而后来补入界限突破的武将又被整体纳入。

本次改为三个按出版阶段递进的面杀预设，并保留完整将池。预设首先按来源包排序，再用极少量人工例外修正明显跨档武将。所有预设使用移动版当前身份局技能。

## 已确认的产品原则

- 四档逐级包含：`经典身份 ⊂ 界限平衡 ⊂ 进阶平衡 ⊂ 完整将池`。
- 以出版阶段为主，不逐个武将进行主观强度评分。
- 旧神将按实际出版阶段进入将池，不因为“神”前缀统一排除。
- 能用实体牌、标记物、纸笔完成的技能直接保留。
- 需要随机、记录或移动版专属小游戏的武将，在提供对应助手后保留。
- 仍依赖无法复现的客户端隐藏状态、且没有可靠助手的武将，不进入任何抽将池。
- 完成后同步到现有 Netlify、GitHub，以及 `https://wujiangtai-hero-draw.liujunyang19.chatgpt.site/`。

## 推荐将池定义

每名武将生成 `presetLevel: 1 | 2 | 3 | 4`。选择某档时，纳入所有 `presetLevel <= 当前档位` 且未标记为不可面杀的武将。

### 1. 经典身份（149 名）

包含：

- 标准包；
- 神话再临风、火、林、山包，包括其中的早期八神；
- 一将成名 2011—2015；
- SP 武将组 1—5。

定位：最接近传统线下身份局，技能短、专属机制少、爆发上限较低。

### 2. 界限平衡（累计 293 名）

在经典身份上增加：

- 界限突破标准、风、火、林、山包；
- 界一将成名 2011—2015；
- 乱世英杰 2016—2017；
- SP 武将组 6—9。

人工例外：`界徐盛` 顺延至进阶平衡。其他武将不因单次技能加强而跨档，避免将池退化为主观评级表。

定位：以界限突破时代为强度上限，适合作为默认推荐档。

### 3. 进阶平衡（累计 400 名）

在界限平衡上增加：

- 神话再临阴、雷包；
- 始计篇仁、信、严、勇、智包；
- SP 武将组 10—12；
- 袖里乾坤武将组 1—3；
- 将星独具武将组 1—2；
- 从较低档顺延的 `界徐盛`。

定位：允许更多专属机制和较高爆发，但不进入谋、势等当前高强度系列。

### 4. 完整将池（573 名，随移动版数据更新）

包含全部可面杀武将。以下较新的来源仅在完整将池出现：

- 谋攻篇、兵势篇；
- SP 武将组 13—15；
- 将星独具武将组 3 与未分组星将；
- 袖里乾坤武将组 4—5、千里之骥、老友季；
- 龙血玄黄、江山如故、君子六艺、缘包；
- 最新、限时、联动、疑、魔及其他未分组武将。

完整将池的数量从数据生成结果读取，不在界面代码中硬编码。

## 面杀兼容分级

每名武将新增：

```ts
faceToFace: "native" | "assisted" | "excluded"
assistantModules: string[]
```

- `native`：无需软件介入，默认值。
- `assisted`：必须存在已实现的助手模块，才可进入抽将池。
- `excluded`：保留数据供检索和说明，但不进入四档抽取结果。

当前数据中的首批 `assisted` 武将：

| 武将 | 模块 | 用途 |
| --- | --- | --- |
| 左慈、界左慈 | 化身助手 | 从未使用武将中随机生成化身，显示可选技能，记录当前化身与已移除化身 |
| 郑玄 | 整经助手 | 复现“整理经典”小游戏并返回获得的牌数结果 |
| 马钧 | 百戏与精械助手 | 复现水转百戏结果，并展示升级装备的当前效果 |
| 周群 | 命运签助手 | 按移动版命运签规则抽签，支持一次合法的暗中加签 |
| 十常侍 | 结党助手 | 管理未亮出的常侍、主将与四名副将候选，以及认可关系 |

当前 573 名数据在上述模块完成后没有必须永久排除的武将。以后数据更新时，如新增移动版专属交互而没有对应模块，必须先标记为 `excluded`，不能默认进入抽将池。

## 助手交互

- 武将详情卡在 `assistantModules` 非空时显示“打开面杀辅助”。
- 助手以全屏抽屉展示，顶部固定武将名、技能名、本局重置按钮和关闭按钮。
- 所有随机结果使用 `crypto.getRandomValues`；需要“不重复”的模块维护本局已使用集合。
- 本局状态只存浏览器本地存储，不需要账号或服务器；关闭页面重新打开后可继续，用户可随时清空。
- 助手不追踪实体牌堆的未知顺序，不替玩家读取或推断隐藏手牌。
- 计数、标记和公开牌名采用通用记录组件；五类专属玩法使用独立配置和规则函数。
- 如果模块数据缺失或规则校验失败，该武将即时视为 `excluded`，界面说明原因，不允许抽中。

## 页面改动

- 将单个“推荐将池”按钮替换为四段预设选择器，显示每档名称和当前人数。
- 默认选择“界限平衡”；“完整将池”仍允许用户再按势力、系列和品质缩小范围。
- 切换预设会清空本轮抽取历史，防止不同档位的“不重复”状态互相污染。
- 当前预设说明显示其定位和新增范围；需要辅助的武将在卡面与详情页显示“需辅助”徽标。
- `excluded` 武将只在按名字搜索时出现，并显示“不可面杀”，不能进入随机抽取结果。

## 数据与代码边界

- `scripts/update-mobile-heroes.mjs` 根据 `sourcePack` 生成基础 `presetLevel`，再应用小型 `tierOverrides` 表。
- 面杀兼容性由独立配置维护，避免把助手规则散落在抓取脚本中。
- 页面只消费生成后的等级和兼容字段，不重复判断来源包。
- 推荐池选择器、通用记录组件和技能助手抽屉拆成独立组件；随机与分档规则保持为无 UI 的纯函数，便于测试。

## 校验与测试

- 固定当前三个累计人数：149、293、400；完整池读取当前数据数量 573。
- 验证四档严格逐级包含，不存在低档缺失、高档反而出现的武将。
- 代表性用例：早期八神进入经典身份；界夏侯惇进入界限平衡；界徐盛进入进阶平衡；谋、势武将仅进入完整池。
- 验证 `assisted` 武将都有已注册模块，`excluded` 武将永远不会被随机抽中。
- 为化身不重复抽取、命运签加签、十常侍认可关系和小游戏重置编写规则测试。
- 构建 Vinext 与 Netlify 静态版本，运行现有数据清洗回归测试。
- 发布后验证 Netlify 和指定 ChatGPT Sites 地址均加载新版预设和技能助手。

## 非目标

- 不制作完整三国杀规则引擎。
- 不扫描或追踪实体牌堆、手牌和身份牌。
- 不按胜率为 573 名武将逐个打分。
- 不增加登录、云同步或多人联网房间。
